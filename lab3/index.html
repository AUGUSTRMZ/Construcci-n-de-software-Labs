Preguntas a responder

¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es la red física de computadoras interconectadas, mientras que la Web es un sistema de documentos (páginas) enlazados que usan esa red.

¿Cuáles son las partes de un URL?

Protocolo (https), dominio (ejemplo.com), puerto (opcional), ruta (/ruta), parámetros (?id=1), fragmento (#sección).

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET obtiene datos, POST envía datos, PUT/PATCH actualizan, DELETE elimina, HEAD obtiene cabeceras.

¿Qué método HTTP se debe usar al enviar un formulario con usuario y contraseña?

POST, porque envía los datos en el cuerpo de la solicitud de forma más segura.

¿Qué método se usa al acceder a una página desde el navegador?

Se usa GET.

¿Qué significa una respuesta HTTP 200?

Que la solicitud fue exitosa y no hubo errores.

¿El desarrollador debe corregir un error 404?

Sí. Es su responsabilidad asegurar que los recursos existen y que los enlaces estén correctos.

¿El desarrollador debe corregir un error 500?

Sí. Es un error del servidor, generalmente por código incorrecto.

¿Qué significa que un atributo HTML5 esté deprecado?

Que ya no se recomienda usarlo porque puede dejar de funcionar. Ej: bgcolor, align, font.

¿Diferencias entre HTML 4 y HTML5?

HTML5 incluye etiquetas semánticas, soporte multimedia, validación de formularios y APIs nuevas.

¿Qué estructura tiene una tabla HTML?

`table`, `thead`, `tbody`, `tr`, `th`, `td`.

¿Cuáles son los controles de una forma HTML5?

`input`, `textarea`, `select`, `button`, `datalist`, `fieldset`, etc.

¿Qué tanto soporte HTML5 tiene tu navegador?

520 puntos de 555

¿Cuál es el ciclo de vida de los sistemas de información?

Planificación, análisis, diseño, desarrollo, implementación, mantenimiento.

¿Cuál es el ciclo de desarrollo de sistemas de información?

Análisis de requisitos, diseño del sistema, desarrollo, pruebas, implementación y mantenimiento.
